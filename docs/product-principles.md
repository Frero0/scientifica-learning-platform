# Product Principles

## Interactive Scientific Learning

Learners should manipulate ideas, answer questions, and receive feedback before passive explanation. The platform should make scientific reasoning visible and testable.

## Problem Solving First

Lessons should quickly move from context to a concrete task. Reading supports action; it does not replace it.

## Visual Explanations

Visuals should clarify relationships between variables, models, and observations. Avoid decorative visuals that do not improve understanding.

## Progressive Paths

Courses are composed of modules, lessons, and exercises. Each level should have a clear prerequisite relationship and a measurable learning outcome.

## Intelligent Gamification

Achievements and progress should reinforce mastery signals: correct reasoning, completed lessons, improved attempts, and durable understanding. Avoid mechanics that reward empty activity.

## Minimal Premium UI

The interface should feel calm, precise, and focused. Use strong spacing, restrained color, readable hierarchy, and high-quality interactions rather than visual noise.
